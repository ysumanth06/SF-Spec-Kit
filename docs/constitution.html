The Core Framework

Architectural Constitution & Core Principles

A rigorous framework ensuring evidence-based quality for AI-driven development. These articles define the boundaries of autonomous execution.

01
account_tree

Article I: Metadata-First

Principle: Objects/Fields before logic

Declarative foundations are the bedrock of the Salesforce platform. Metadata structures must be finalized and validated before any Apex orchestration begins.

02

Article II: Bulk Awareness

Mandatory 201+ record handling for all operations. AI must prove bulk-safety in all generated triggers and service logic.

03

Article III: Declarative-First

Flow over Apex decision mandate. Code is only permitted when declarative tools cannot meet the architectural requirements.

Article IV: Absolute Security

Strict enforcement of `with sharing` and `WITH USER_MODE`. No data leakage is permitted within the execution spectrum.

Article V: PNB Test Pattern

Mandatory Positive, Negative, and Bulk test scenarios for every unit of unit of deployment.

Article VI: Clean Layers

Service, Selector, Domain

Strict separation of concerns. AI agents must follow the domain-driven architecture for maintainable, reusable logic units.

Article VII: Deployment Safety

Mandatory Dry-runs

No release is valid without an automated proof of compliance and dependency validation across environments.

Article VIII: Platform Context

Every prompt and execution must be tethered to real-time architectural clarity. No assumptions are permitted in the Build phase.

Article IX: Modular Logic

Reusable, testable domain units. Code must be atomized to prevent technical debt and simplify future AI refactoring.
